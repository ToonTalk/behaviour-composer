I change the initial geometry of the world to be a torus. In other words, when an individual moves off the edge it appears on the opposite edge as if the surface has been rolled into a tube and then the ends of the tube connected.

Begin micro-behaviour

SET-WORLD-TO-TORUS

Begin NetLogo code: 
set-world-geometry 1
End NetLogo code

Variants

set-world-geometry expects one number which can have any of the following values:

  1. Torus (wraps horizontally and vertically) SET-WORLD-TO-TORUS
  2. Vertical cylinder (wraps horizontally) SET-WORLD-TO-VERTICAL-CYLINDER
  3. Horizontal cylinder (wraps vertically) SET-WORLD-TO-HORIZONTAL-CYLINDER
  4. A plane (no wrapping -- objects become invisible if beyond the limits of the world) SET-WORLD-TO-PLANE
  5. A plane with a camera that follows the average position of all individuals SET-WORLD-TO-PLANE-WITH-CAMERA-FOLLOWING-CENTROID

How this works

Due to current limitations of NetLogo this specifies only the initial geometry of the world and cannot be used to change the world's geometry during execution.
