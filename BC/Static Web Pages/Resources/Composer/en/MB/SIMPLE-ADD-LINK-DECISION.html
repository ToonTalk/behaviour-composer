I decide whether to form a link with another player.

Begin micro-behaviour

SIMPLE-ADD-LINK-DECISION

Begin NetLogo code: 
do-after .001
   [do-if is-object? my-other-player and 
          not member? my-other-player my-player-links
       [set my-add-link-utility-gain 
            utility-of-network (list my-other-player) [] -
            utility-of-network [] []
        set my-desire-to-link my-add-link-utility-gain > 0 
        add-behaviour ADD-A-LINK-IF-IN-AGREEMENT]]
End NetLogo code

Related Micro-behaviours

ADD-LINK-DECISION-ERROR can reverse the outcome of this micro-behaviour. SIMPLE-REMOVE-LINK-DECISION decides whether to remove an existing link. This uses UTILITY-OF-NETWORK to compute the utility of the current network and the network with some additions and removals. LOG-DESIRE-TO-ADD-A-LINK adds to the log a report when this sets my-desire-to-link to true.

How this works

If my utility increases by adding the player I'm paired with then record my desire to link with that player. I then add ADD-A-LINK-IF-IN-AGREEMENT that will check if the other also decided to link and if create the link. This also records my-add-link-utility-gain which can be used to gather statistics.
