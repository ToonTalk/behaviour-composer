I repeatedly move to a nearby position that is unoccupied (or run forever if there are none).

Begin micro-behaviour

WANDER-TO-RANDOM-LOCATION-UNTIL-EMPTY

Begin NetLogo code: 
do-every delta-t
   [let unoccupied-location random-location-found-to-be-unoccupied (xcor - 1) (xcor + 1)
                                                                   (ycor - 1) (ycor + 1)                                                                  
                                                                   1000000
    set my-x first unoccupied-location
    set my-y second unoccupied-location]
End NetLogo code

Variants

The 1s in the code specify a range of movement plus or minus one unit horizontally and vertically. You can use other numbers to specify a different range. You can specify any region of the world by specifying the minimum x, maximum x, minimum y, and maximum x. This calls random-location-found-to-be-unoccupied tries random coordinates until an unoccupied location is found or it gives up if 1000000 attempts fail to find an unoccupied spot.

Related Micro-behaviours

WANDER-TO-AN-UNOCCUPIED-LOCATION behaves the same as this micro-behaviour if there are unoccupied locations in the region. It takes more computation but will not loop forever if the region is full.

WANDER-RANDOMLY-IN-COMPASS-DIRECTIONS moves in a similar fashion but allows movement to occupied locations. SET-RANDOM-UNOCCUPIED-POSITION moves to any unoccupied location.

How this works

It uses random-location-found-to-be-unoccupied to compute a list of all unoccupied locations within the coordinate range and selects a random element. If there are no unoccupied locations this micro-behaviour never stops.
