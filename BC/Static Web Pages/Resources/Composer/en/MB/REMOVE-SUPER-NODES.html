I remove individuals with the greatest number of acquaintances.

Begin micro-behaviour

REMOVE-SUPER-NODES

Begin NetLogo code: 
do-at-time .000002
   [let population-sorted sort-by [[count my-acquaintances] of ?1 > [count my-acquaintances] of ?2] all-living
    repeat the-number-of-super-nodes-removed 
           [let super-node first population-sorted
            set population-sorted butFirst population-sorted
            ask super-node [output-print (word "Removed a node with " count my-acquaintances " acquaintances")]
            ask [my-acquaintances] of super-node
                [set my-acquaintances my-acquaintances with [self != myself]]
            add-behaviour-to super-node DIE]]
End NetLogo code

Variants

Rather than remove these nodes during model setup you can change the .000002 to be a later time (e.g., to model an intervention).

Related Micro-behaviours

CREATE-NUMBER-OF-SUPER-NODES-SLIDER  controls the number of nodes removed. INITIALISE-SOCIAL-NETWORK-NORMAL-DISTRIBUTION-SYMMETRIC and INITIALISE-SOCIAL-NETWORK-POWER-LAW-DISTRIBUTION-SYMMETRIC create non-uniform social networks.

How this works

This sorts the set of all living by the number of acquaintances. Then for each one removed it updates my-acquaintances of its acquaintances to no longer include the removed individual. This is scheduled to happen immediately after INITIALISE-SOCIAL-NETWORK-NORMAL-DISTRIBUTION-SYMMETRIC or INITIALISE-SOCIAL-NETWORK-POWER-LAW-DISTRIBUTION-SYMMETRIC creates the network.
