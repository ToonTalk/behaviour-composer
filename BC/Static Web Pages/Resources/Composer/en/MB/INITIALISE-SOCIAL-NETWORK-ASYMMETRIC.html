I initialise my social network with a constant number of acquaintances. A link from me to another does not mean there is a link back to me.

Begin micro-behaviour

INITIALISE-SOCIAL-NETWORK-ASYMMETRIC

Begin NetLogo code: 
do-after-setup
   [set my-acquaintances 
        n-of minimum count all-individuals 5 
             all-individuals]
End NetLogo code

Variants

You can change the 5 to any other number to initialise the subset of the "objects" (or agents) that I know. This can have a distribution of different number of acquaintances by replacing the "5" with something like "round within-range (random-normal 5 2) 1 10 " to create a distribution with a mean of 5 and a standard deviation of 2 truncated to values between 1 and 10 inclusive.

Or you may wish to replace the "5" with a variable defined by a slider for easy experimentation. See CREATE-SLIDER.

Related Micro-behaviours

INITIALISE-SOCIAL-NETWORK-NORMAL-DISTRIBUTION-ASYMMETRIC is similar to this one except the number of acquaintances is normally distributed. An example of the use of this micro-behaviour is CHANGE-OPINION-SOCIAL-NETWORK.

How this works

It randomly selects 5 individuals as my-acquaintances. It is careful to avoid creating more acquaintances than there are individuals by taking the minimum of 5 and the number of individuals. This code is run after setup has completed since it is typically during setup that copies are created. round is needed to convert to an integer value.
