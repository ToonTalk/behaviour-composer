A Guide to Reconstructing Sugarscape

in the BehaviourComposer

Sugarscape is an artificial world with life, death, wealth, sex, culture, trade, and disease described in Growing Artificial Societies by Joshua M. Epstein and Robert Axtell, Brookings Institute Press, The MIT Press, 1996. Here we focus upon life, death, and wealth from chapter 2.

This guide to building Sugarscape was constructed by Ken Kahn.

6 April, 2011 Version

Building Sugarscape

Step 1. Building the environment

You will start with a model of the Sugarscape where a few technical details have been taken care of. To load this model click the Initial Sugarscape Model button and choose 'Replace current model' from the menu. You will see that two agents have been created: 'Agent' and 'Sugarscape Environment'. The Sugarscape Environment has been created with a number of NetLogo code fragments called behaviours. You will build on this model by adding more behaviours from the Sugarscape Library.

The environment needs to be initialised to give it the behaviour of producing sugar. To do this you need to add the following behaviours to Sugarscape Environment agent:

  • INITIALISE-SUGAR-CAPACITY to the Sugarscape Environment so that each patch of the Sugarscape has a maximum sugar capacity,
  • INITIALISE-SUGAR-GROWTH-RATE to set the rate in which sugar grows,
  • GROW-SUGAR-DELAYED to grow the sugar, and
  • UPDATE-PATCH-COLOUR so the patches are coloured to reflect the amount of sugar accumulated.

To do this click on each of the links above in turn to go to each behaviour's web page. Then click on the button at the top of each behaviour's web page (e.g. the button titled 'INITIALISE-SUGAR-CAPACITY'), select 'Add to prototype' from the menu, then within 'Composer' area click within the Sugarscape Environment box to drop a behaviour onto this specific 'agent' or 'prototype'.

Next click on the 'Run your model' tab within the Composer Area and then click the 'GO' button within the Applet. The Sugarscape Environment should be initialised and quickly grow sugar up to the capacity of each patch; it should look like this after a few seconds:

Suggestion: You should get used to reading what is written on each behaviour web page and trying to follow how the behaviour works.

Step 2. Creating an Agent

If you run your model and look carefully you'll see the you already have an Agent in your model - a small triangle in the bottom-left corner. Now you will make this agent behave as in chapter 2 of the Growing Artificial Societies book. First go to the 'Sugarscape Agents' section within the Sugarscape Library, find the 'Initialise Agent' behaviour and add it to Agent within the Composer area (following a similar process as in the previous step). Now go through the same steps to add behaviours that put the agent in an 'Initial random position'.

Finally add behaviours that set the 'shape' and 'colour' of the Agent (you will find these in the Appearance and Movement sections of the Library). You can change the shape and colour from their defaults by opening the behaviours and changing the parameters within the NetLogo code on the page (for instance you might want to change the parameter "fish" to "person" in the shape behaviour).

Step 3. Make the Agent Move

Add a behaviour to make Agent move to the nearest patch with the most sugar that it can see. This behaviour also requires you to add a behaviour that gives an Agent a preference for patches with more sugar.

When you run the model, you may see the Agent move a little bit before it stops. You can produce different trials by clicking on the 'Setup' button followed by the 'Go' button.

Suggestion: Think about why the agent sometimes doesn't move at all.

Step 9. Add a Lorenz Curve and a Monitor for the Gini Coefficient

The Lorenz Curve and Gini Coefficient are additional tools for observing the distribution of wealth in the Sugarscape. (See page 36 of Growing Artificial Societies.) Add behaviours that monitor these values. (Note you will need to add two behaviours to create the plotting area and to plot the graphs.) Also a monitor that displays the current Gini Coefficient.

Step 10. Add Monitors for Vision Level and Sugar Metabolism

Now add monitors for Vision Level and Sugar Metabolism.

Step 11. Run some Experiments

You can use the sliders in the model as it runs in the Applet to explore the parameter space of the Sugarscape model. (Using models in this way is sometimes called in silico experimentation).

Run experiments to discover how to make large and small Gini coefficients and see which factors influence the carrying capacity (the stable size of the population i.e. you can read this from the graph at time 50).

Use this form to record your findings and answer the questions. Your answers will be recorded in an online spreadsheet (Google Docs service) so that we can discuss this step later on in the class.

Suggestion: You can explore the stochastic nature of the model by making repeated runs with the same parameters.

Step 12. Replacing dying agents

Add a behaviour so new agents are created when an agent dies.

Suggestion: Change the initial population to 250 and observe how death and rebirth affects the dynamics of the mean vision level, the mean sugar metabolism, and the wealth distribution.

Step 13. Limited Lifetimes

Give the agents behaviours so they die when they reach their lifespan.

Suggestion: See how this affects the dynamics of the mean vision level, the mean sugar metabolism, and the wealth distribution. Note that the way aging is modelled will begin to have an effect after 60 time steps.

Step 14. Production Causes Pollution

Add behaviours so that sugar production causes pollution and agents move to avoid polluted areas.

Suggestion: Observe the modified dynamics. Note that the production of pollution doesn't begin before 50 time steps. Sugar production is done by the Sugarscape Environment.

Step 15. Consumption Causes Pollution

Add behaviours so that sugar consumption causes pollution. This also begins after 50 time steps. Sugar consumption is done by each agent.

Step 16. Diffusing Pollution

Add a behaviour to the Sugarscape Environment so that after 100 time steps pollution starts to diffuse to neighbouring sites.

Step 18. If you have time

If you have time you could also:

  • Replace the behaviour for growing sugar with one that has differential seasonal growth rates.
  • Replace the initial distribution of the population with one where everyone starts in the Southwest corner.
  • Further explore the parameter space.
  • Also NetLogo has a tool for automating the exploration of the parameter space. You can download NetLogo and run your model there. Click on Tools and BehaviorSpace to try it out.
