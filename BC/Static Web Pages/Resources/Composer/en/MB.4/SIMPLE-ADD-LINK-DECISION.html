I decide whether to form a link with another player.

Begin micro-behaviour

SIMPLE-ADD-LINK-DECISION

Begin NetLogo code: 
do-if not link-neighbor? my-other-player and count link-neighbors < the-maximum-number-of-links
   [set my-add-link-utility-gain 
        utility-of-network (list my-other-player) [] -
        utility-of-network [] []
        set my-desire-to-link my-add-link-utility-gain > 0]
End NetLogo code

Related Micro-behaviours

ADD-LINK-DECISION-ERROR can reverse the outcome of this micro-behaviour. SIMPLE-REMOVE-LINK-DECISION decides whether to remove an existing link. This uses UTILITY-OF-NETWORK to compute the utility of the current network and the network with some additions and removals. LOG-DESIRE-TO-ADD-A-LINK adds to the log a report when this sets my-add-link-utility-gain to a positive value.

How this works

This records my utility increases by adding the player I'm paired with as my-add-link-utility-gain. Another micro-behaviour should add ADD-A-LINK-IF-IN-AGREEMENT that will check if both players have true values for my-desire-to-link and if so creates the link. If the players are already linked this does nothing.
