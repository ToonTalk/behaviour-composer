I vaccinate some number of susceptible individuals after a delay.

Begin micro-behaviour

VACCINATE-N-RANDOM-INDIVIDUALS

Begin NetLogo code: 
do-after the-time-of-vaccination - 0.001
   [do-for-n the-vaccine-count all-susceptibles
       [do-with-probability the-fraction-of-vaccinations-that-take
           [add-behaviours list-of-micro-behaviours "Vaccinated Behaviours" [SET-STATE-VACCINATED.html]]]]
End NetLogo code

Variants

The do-after command could be replaced by a when command that specifies a condition, e.g. when the number of infected individuals exceeds a threshold.

Related Micro-behaviours

CREATE-VACCINE-COUNT-SLIDER controls the-initial-vaccine-count. Slider for time of vaccination is customised to control the-time-of-vaccination.

This micro-behaviour relies upon SET-STATE-VACCINATED to set the state of the copies to "vaccinated".  REPORT-ALL-SUSCEPTIBLES computes the set of individuals who are susceptible to infection.

VACCINATE-N-RANDOM-SUSCEPTIBLE-ACQUAINTANCES-OF-THE-INFECTED differs from this micro-behaviour in that it restricts its actions to those individuals who are susceptible and have at least one acquaintance who is infected.

How this works

This selects the-vaccine-count from the population of all susceptible individuals and adds the SET-STATE-VACCINATED micro-behaviour to each of them with a probability of the-fraction-of-vaccinations-that-take. The actions are delayed by the value of the-time-of-vaccination which has a tiny number subtracted from it in order to ensure that this runs before the next encounter event.

do-for-n selects n agents and do-with-probability does the action probabilistically.
