I sometimes flip the decision as to whether to remove a link.

Begin micro-behaviour

REMOVE-LINK-DECISION-ERROR

Begin NetLogo code: 
do-with-probability the-odds-of-remove-link-error
   [if-else my-decision-to-unlink = true 
       [set my-decision-to-unlink false]
       [if my-decision-to-unlink = false 
           and ; I'm linked to the other player
           link-neighbor? my-other-player 
           [set my-decision-to-unlink true]]]
End NetLogo code

Related Micro-behaviours

REMOVE-LINK-ERROR-ODDS-SLIDER adds a slider for the-odds-of-remove-link-error. This adds noise (a decision error) to SIMPLE-REMOVE-LINK-DECISION. ADD-LINK-DECISION-ERROR adds to the decision to add links.

How this works

After SIMPLE-REMOVE-LINK-DECISION has set my-decision-to-unlink this probabilistically flips the decision. It runs before REMOVE-A-LINK-IF-DECIDED looks the values of my-decision-to-unlink.

History
