I copy the value of one of my attributes to the attribute of another.

Begin micro-behaviour

COPY-ATTRIBUTE

Begin NetLogo code: 
substitute-text-area-for columns: 70 my-attribute-name my-x
substitute-text-area-for columns: 71 other-attribute-name my-x
substitute-text-area-for columns: 76 other-agent the-other     
substitute-text-area-for columns: 80 rows: 5 comment ; Notes                                                                                      
let value my-attribute-name
ask other-agent
    [set other-attribute-name value]
comment
End NetLogo code

Related micro-behaviours

UPDATE-ATTRIBUTE updates an attribute. DEFINE-PARAMETER creates a model parameter (while this micro-behaviour creates one whose value can differ among agents).  UPDATE-PARAMETER updates parameters.

Variants

Change the attribute name in the first text areas whatever attribute you want to create or set. And second text area can contain any value or an expression that will evaluate to another agent or set of agents.

How this works

The BehaviourComposer treats specially variables whose name begins with "my-" or "my-next-". Typically micro-behaviours set the "my-next-" version and read from the "my-". Whenever the time advances the "my-" variable gets the value of the "my-next-" variable. This fetches the value of my attribute and sets the attribute of another to that value.

Note that, unlike NetLogo, you don't need to declare "breed" variables.
