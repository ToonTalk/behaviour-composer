I initialise the rate in which I consume sugar.

Begin micro-behaviour

INITIALISE-SUGAR-CONSUMPTION

Begin NetLogo code: 
set my-sugar-consumption random-integer-between the-minimum-sugar-consumption the-maximum-sugar-consumption
End NetLogo code

Variants

Sugar consumption need not be selected uniformly from the interval [the-minimum-sugar-consumption, the-maximum-sugar-consumption] but can be real numbers or have other distributions.

Related Micro-behaviours

CONSUME-SUGAR-DELAYED is the process that uses my-sugar-consumption to implementing consumption. INITIALISE-AGENT adds this micro-behaviour.

How this works

This sets my-sugar-consumption to a random integer between the-minimum-sugar-consumption and the-maximum-sugar-consumption inclusive.

Source

This is based upon page 24 of [1]. The book is silent as to whether this needs to be an integer or can be any number between 1 and 4.
