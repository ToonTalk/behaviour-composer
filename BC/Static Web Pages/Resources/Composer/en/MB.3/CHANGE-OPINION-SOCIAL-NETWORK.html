I repeatedly select a random acquaintance and change my opinion accordingly. This implements part of the model in [1].

Begin micro-behaviour

CHANGE-OPINION-SOCIAL-NETWORK

Begin NetLogo code: 
do-every 1
   [set-the-other one-of my-acquaintances
    add-behaviours list-of-micro-behaviours "Behaviours added to me" [RELATIVE-AGREEMENT.html]
    add-behaviours-to the-other list-of-micro-behaviours "Behaviours added to other" [RELATIVE-AGREEMENT.html]]
End NetLogo code

After every time unit then I run the RELATIVE-AGREEMENT micro-behaviour with a randomly selected acquaintance which may change my opinion. My acquaintance also runs this micro-behaviour.

Variants

To see a log of the changes in opinion and uncertainty add copies of the ADD-MESSAGE micro-behaviour with these messages:

(word "self: " who " opinion: " my-opinion " next-opinion: " my-next-opinion)
(word "other: " who-of the-other " uncertainty: " my-uncertainty)
SPECIFY-OUTPUT-AREA specifies up the dimensions and placement of the output area.

Related Micro-behaviours

CHANGE-OPINION-IN-NEIGHBOURHOOD differs from this in that the interaction is with someone located spatially nearby rather than an acquaintance anywhere in the model. This relies upon RELATIVE-AGREEMENT and INITIALISE-SOCIAL-NETWORK-CONSTANT-ASYMMETRIC.  It uses the-other to refer to another agent.
