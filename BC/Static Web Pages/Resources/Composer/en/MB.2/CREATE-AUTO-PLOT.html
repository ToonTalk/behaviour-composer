I set up and maintain a plot of two variables, e.g. hunger versus elapsed time. The location and size are specified but the minimum, and maximum values are calculated dynamically.

Begin micro-behaviour

CREATE-AUTO-PLOT

Begin NetLogo code: 
substitute-text-area-for lower-left-corner-x 5    
substitute-text-area-for lower-left-corner-y 550 
substitute-text-area-for upper-right-corner-x 420  
substitute-text-area-for upper-right-corner-y 809 
substitute-text-area-for plot-label Hunger Plot     
substitute-text-area-for x-axis-label time        
substitute-text-area-for y-axis-label hunger      
substitute-text-area-for x-axis-value time               
substitute-text-area-for y-axis-value my-hunger          
create-plot
  "lower-left-corner-x" "lower-left-corner-y" ; lower left corner 
  "upper-right-corner-x" "upper-right-corner-y" ; upper right corner
  "plot-label" ; unique name (and label) for this plot
  "x-axis-label" ; X axis label
  "y-axis-label" ; Y axis label
  x-axis-value ; X axis value
  y-axis-value ; Y axis value
End NetLogo code

Variants

You can plot other attributes. You need not have time as the x-axis.

You don't need to specify the location of the plot if the default is acceptable. Also you needn't specify the X and Y values since they can be handled by other micro-behaviours. This is the best way to have multiple graphs on the same plot. See, for example, PLOT-MY-OPINION.

This uses the NetLogo auto-plot feature to rescale the plot as needed. Use CREATE-PLOT if you wish to specify the minimum and maximum values.

You can replace my-hunger with other attributes. For example, use 

count all-individuals with [kind = "fish"]
my-hunger to graph the fish population.

Related Micro-behaviours

CREATE-OPINION-PLOT and PLOT-AVERAGE-DIRECTION are two other examples of creating plots. CREATE-HISTOGRAM creates histograms.
