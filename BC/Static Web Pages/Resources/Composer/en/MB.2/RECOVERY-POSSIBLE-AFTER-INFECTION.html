If I become infected I add a process that models recovery.

Begin micro-behaviour

RECOVERY-POSSIBLE-AFTER-INFECTION

Begin NetLogo code: 
when [my-state = "infected"]
     [do-after 1.01 [add-behaviours list-of-micro-behaviours "Possible Recovery Behaviours" [POSSIBLY-RECOVER.html]]]
End NetLogo code

How this works

When SET-STATE-INFECTED runs it sets my-state to infected. If this micro-behaviour has also been added then this will run after that and add the POSSIBLY-RECOVER process. This delays adding the micro-behaviour since you can't recover just after getting infected. The delay is 1.01 rather than 1.0 so that this process runs after the processes that model infectious encounters.
