This delays the execution of a list of micro-behaviours.

Begin micro-behaviour

DO-AFTER

Begin NetLogo code: 
substitute-text-area-for delay-duration 1                                           
substitute-text-area-for comment Notes                                               
do-after delay-duration 
  [add-behaviours 
       list-of-micro-behaviours "Delayed Behaviours" []]
;comment
End NetLogo code

Related Micro-behaviours

DO-AFTER-SETUP adds behaviours after setup.

Variants

You can edit the duration field to any positive number. You can add micro-behaviours to the list of behaviours.

How this works

This relies upon the Behaviour Composer scheduling extensions to NetLogo to run the list of micro-behaviours after a delay.

History

DO-AFTER was implemented by Ken Kahn on 31 July 2009.
