I compute the appropriate growth factor for the season of each patch.

Begin micro-behaviour

COMPUTE-SEASONAL-GROWTH-FACTOR

Begin NetLogo code: 
substitute-text-area-for season-length-edit-area 50 
substitute-text-area-for winter-growth-factor-edit-area 1 / 8  

to-report seasonal-growth-factor 
   let season-length season-length-edit-area
   let winter-growth-factor winter-growth-factor-edit-area
   let northern-hemisphere (pycor >= max-pycor / 2)
   ifElse time mod (season-length * 2) > season-length
     [ifElse northern-hemisphere [report 1] [report winter-growth-factor]]
     [ifElse northern-hemisphere [report winter-growth-factor] [report 1]]
end
End NetLogo code

Variants

You can experiment with a different season-length or winter-growth-factor.

How this works

This adds to your model a new NetLogo reporter that can be used by any micro-behaviour. Here it defines seasonal-growth-factor that computes the growth factor of each patch in the model. Every season-length time units it switches factors between the southern and northern halves of Sugarscape. It is used by GROW-SUGAR-SEASONALLY.
